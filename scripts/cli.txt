hnote 크롤러 CLI 가이드
=====================

모바일 앱에서 저장한 네이버 부동산 URL(pending)을 PC에서 크롤합니다.


사전 준비
---------

1. 프로젝트 루트에 .env 설정

   VITE_SUPABASE_URL=
   SUPABASE_SERVICE_ROLE_KEY=     # PC 워커 권장 (없으면 publishable key)

2. Lightpanda 설치 (기본 경로: ~/.local/bin/lightpanda)

   다른 경로면 LIGHTPANDA_PATH=/path/to/lightpanda


자주 쓰는 명령
--------------

# pending 1배치 처리 후 종료
bun run crawl:once

# pending 없을 때까지 5초마다 폴링 (Ctrl+C 종료)
bun run crawl

# 직접 실행 (옵션 붙일 때)
bun run scripts/crawl.ts --once
bun run scripts/crawl.ts

# URL 직접 크롤 (DB 반영 없이 미리보기)
bun run scripts/crawl.ts --url "https://naver.me/xk9FceFI"

# URL 크롤 + 기존 매물 DB 반영
bun run scripts/crawl.ts --url "https://fin.land.naver.com/map?..." --property-id <uuid>


옵션
----

--url <url>         URL 직접 크롤. --property-id 없으면 미리보기만 (Supabase 불필요)
--property-id <id>  --url 과 함께: 해당 매물에 URL·크롤 결과 반영
# 2) complex_detail  단지+필터에서 공유 (예: naver.me/xk9FceFI)
#    → layer에서 complexId·평형·거래유형 추출
#    → RSC GET /complex 로 단지 메타 저장 (기본)
#    → 실거래는 RSC·API 보조 (429 시 메타만)
#
#    --resolve-article / --pick-first 를 쓰면 예전처럼 article_detail 매물 1건으로 변환

--resolve-article   complex_detail URL을 article_detail 매물 1건으로 변환 (2차 Lightpanda)
--pick-first         --resolve-article 과 함께: 후보 여러 건일 때 최저가 1건 자동 선택
--once              1배치만 처리하고 종료
--actor-id <uuid>   특정 저장자(아빠/엄마) 큐만 처리. 생략 시 전체
--batch-limit N     배치당 처리 건수 (기본 20)
--interval-ms N     폴링 간격 ms (기본 5000, 최소 1000)
--wait-ms N         Lightpanda 페이지 대기 ms (기본 40000, 최소 5000)


예시
----

# URL만 테스트 (성공/실패·제목·가격 확인)
bun run scripts/crawl.ts --url "https://naver.me/xxxx"

# naver.me / fin.land 공유 URL — 두 가지 형식 모두 지원
#
# 1) article_detail  매물 카드에서 공유
#    → Lightpanda 1회로 바로 크롤
#
# 2) complex_detail  단지+필터에서 공유 (예: naver.me/xk9FceFI)
#    → 단지 메타 + 최근 실거래 (가능할 때) — 기본 동작
#
# 단지+필터 공유 URL (complex_detail) — 단지 데이터로 저장
bun run scripts/crawl.ts --url "https://naver.me/xk9FceFI"

# 예전 방식: 매물 1건으로 변환 후 크롤
bun run scripts/crawl.ts --url "https://naver.me/xk9FceFI" --resolve-article --pick-first

# 실패한 매물에 새 URL 넣고 재크롤
bun run scripts/crawl.ts --url "https://fin.land.naver.com/map?..." --property-id 6bb5cf1a-921f-46ab-b60e-9212206277be

# 아빠 저장자만, 5건씩 1회
bun run scripts/crawl.ts --once --actor-id <uuid> --batch-limit 5

# 폴링 간격 10초
bun run scripts/crawl.ts --interval-ms 10000


동작 흐름
---------

1. Supabase에서 pending 목록 조회
2. 항목별 processing → Lightpanda 크롤 → completed / failed
3. 터미널에 배치·진행·성공/실패 요약 출력


출력 읽는 법
------------

[배치 #N] 대기 3건 · 이번 처리 3건   ← 큐에 남은 pending / 이번에 돌릴 건수
[1/3] 처리 중 ...                    ← 현재 항목
[1/3] 완료 ... (images=12 · 8421ms) ← 성공
[1/3] 실패 ...                       ← 실패 (앱에서 「다시 시도」 가능)
  ↳ 배치 #N 종료 · 성공 2 · 실패 1
── 세션 요약 ──                       ← --once 종료 시 또는 Ctrl+C 전 누적
